## Format convention

ADRs that touch the database schema include structured tags at the end of their description:

- **Tags**: `data-model` for any ADR that adds/modifies/removes tables or columns.
- **Touches**: `table(column)` listing affected schema objects. E.g., `accounts(dispatch_mode), migration 060`.
- **Relates to**: cross-references to other ADRs on the same concept. E.g., `D37, D96`.
- **Supersedes**: when an ADR replaces a previous decision's schema placement (not its intent).

These tags are mandatory for new data-model ADRs. Retroactively applied to existing ones (WS2, 2026-04-16).
